# GOLDEN_RULES

## 버전
- 패치 파일명은 testV0, testV1, testV2 순서로 올린다.
- Netlify 업로드 주소는 https://app.netlify.com/drop 를 함께 제공한다.

## 탭
- 현재 탭은 m계산 하나만 사용한다.
- 다른 탭 이름은 아직 추가하지 않는다.

## 페이지
- 처음 실행 시 작성중인 페이지가 없으면 + 페이지 버튼만 표시한다.
- + 페이지 버튼은 항상 마지막 슬라이드에 유지한다.
- + 페이지를 눌러 유형을 고르면 새 페이지가 생성되고, 그 옆에 다시 + 페이지가 유지된다.
- 현재 유형은 지하주차장만 사용한다.

## 날짜/달력
- 날짜 기준은 YYYY-MM-DD 형식이다.
- 달력은 왼쪽 상단 달력 아이콘을 눌러 팝업으로 연다.
- 달력의 빈칸은 이전달/다음달 날짜로 채우고 반투명 표시한다.
- 일요일 숫자는 빨간색 계열로 표시한다.
- 오늘 날짜는 숫자 주변에 원 표시한다.
- 달력 칸은 좌우 폭보다 상하 높이를 크게 잡아, 날짜 밑에 추후 표시/메모/이벤트를 넣을 공간을 확보한다.
- 날짜 숫자 자체를 과하게 키우지 않는다.

## 계산
- 총 m = 엔드 - 시작.
- 총 m 칸은 자동 계산 전용으로 직접 입력하지 않는다.

## 저장 구조 예정
- 저장 단위는 날짜별 → 페이지별 → 섹션별 → 행별이다.
- 서버 연결 전에는 UI 확인용으로만 동작한다.

## testV2 반영
- 달력 날짜 숫자는 작게 유지한다.
- 달력 칸은 좌우 폭보다 세로 높이를 더 크게 확보한다.
- 팝업창 높이는 달력 칸 확대에 맞춰 더 길게 잡는다.

## testV3 반영
- 달력 팝업 우측 상단에 X 닫기 버튼을 둔다.
- 오늘 날짜/선택 날짜는 칸 전체가 아니라 숫자 주변 작은 박스만 채운다.
- 날짜 칸은 모두 실선 테두리로 구분한다.

## testV4 반영
- 오늘 날짜 표시는 숫자 주변 테두리만 표시하고 내부 색은 채우지 않는다.
- 달력 팝업은 배경 블러, 둥근 카드, 헤더 pill, 부드러운 실선 그리드로 더 세련되게 정리한다.

## testV5 반영
- 현재 날짜/선택 날짜는 색 채움 없이 숫자 주변 테두리만 표시한다.
- 월 이동 버튼과 닫기 X 버튼이 붙지 않도록 우측 닫기 버튼 공간을 별도로 확보한다.
- 달력 헤더는 좌측 이전, 중앙 월, 우측 다음, 최우측 닫기 구조로 간격을 분리한다.

## testV6 반영
- 오늘 날짜는 숫자 아래 작은 파란 점으로만 표시한다.
- 선택 날짜만 숫자 주변 파란 테두리 네모로 표시한다.
- 오늘 날짜와 선택 날짜의 시각 표현을 분리해 헷갈리지 않게 한다.

## testV7 반영
- 오늘 날짜 표시 점을 숫자 아래가 아니라 숫자 위쪽으로 이동한다.

## testV8 반영
- 오늘 날짜 점을 숫자와 겹치지 않게 더 위로 올린다.

## testV9 반영
- 전체 화면 확대/축소를 막기 위해 viewport에 maximum-scale/minimum-scale/user-scalable=no를 적용한다.
- iOS 제스처 확대와 더블탭 확대를 방지한다.
- 입력칸 폰트는 16px 이상으로 유지해 iOS 입력 시 자동 확대를 줄인다.

## testV10 반영
- 지하주차장 기존 UI를 전부 제거하고 새 구조로 교체한다.
- 상단 묶음은 구분(지상/지하), 층수, 구역, + 버튼으로 구성한다.
- 층수는 숫자 입력이며 칸 오른쪽에 '층'을 표시한다.
- 구역은 텍스트 입력이다.
- 상단 + 버튼은 층수/구역부터 PK 라인까지 한 묶음을 추가한다.
- 배관은 드롭다운이며 소화/위생/급수/오배수/조경을 선택한다.
- 판넬번호는 숫자 입력이며 칸 오른쪽에 '번 판넬'을 표시한다.
- 배관 라인의 + 버튼은 층수/구역은 빼고 배관/판넬/PK 라인 묶음을 같은 구역 아래에 추가한다.
- PK번호는 숫자 입력이며 칸 왼쪽에 'PK'를 표시한다.
- 시작/끝/총은 m 단위 표시를 칸 오른쪽에 둔다.
- 총 = |시작 - 끝| 으로 자동계산하며 +/− 부호는 표시하지 않는다.
- 시작/끝 값은 blur 시 콤마 포맷을 적용한다.
- 총 칸을 누르면 PK번호/시작/끝/총 라인만 추가한다.

## testV11 반영
- 하단 '+ 구역 추가' 버튼은 제거한다.
- '+ PK 추가' 문구는 '+ 추가'로 단순화한다.
- 구역/배관 추가 버튼은 큰 검정 버튼 대신 작은 소프트 아이콘 버튼으로 정리한다.
- 구역/배관 삭제 버튼을 작은 빨간 소프트 아이콘 버튼으로 추가한다.
- 삭제는 최소 1개 묶음은 유지하고 2개 이상일 때만 제거한다.

## testV12 반영
- 구획용 큰 네모 안에 다시 큰 네모가 겹쳐 보이는 구조를 줄이기 위해 구역 상단 박스와 배관 하단 박스를 한 묶음처럼 이어지게 정리한다.
- 바깥 garage-bundle 테두리는 제거하고, 상단/하단 카드가 직접 구획을 만들게 한다.
- PK번호/시작/끝/총 라인 오른쪽에 삭제 버튼을 추가한다.
- PK 라인 삭제는 최소 1개 라인은 유지하고 2개 이상일 때만 제거한다.

## testV13 반영
- 구역 전체를 하나의 큰 카드로 보이게 통합한다.
- 구분/층수/구역 라인은 큰 카드 안의 제목 영역처럼 연한 배경으로 분리한다.
- 배관/판넬번호 라인은 부제목 영역처럼 더 옅은 배경과 구분선으로 분리한다.
- PK번호/시작/끝/총 라인은 표 영역처럼 아래쪽 내용 영역에 배치한다.
- 박스 안 박스 느낌을 줄이고, 한 묶음 안에서 영역만 선/배경으로 나뉘게 정리한다.

## testV14 반영
- 입력칸 위 반복되는 라벨 글자는 제거한다.
- 대신 각 영역 상단에 작은 섹션 타이틀(구역 설정 / 배관 정보 / PK 라인)만 둔다.
- 구역/배관/PK 영역마다 배경 톤과 선 스타일을 다르게 적용해 구획 느낌을 강화한다.
- PK 라인은 개별 카드 느낌으로 살짝 띄워 구분한다.

## testV15 반영
- 값 입력칸 내부가 나뉘어 보이지 않게 단위 표시를 자연스럽게 복구한다.
- 구역 설정 / 배관 정보 / PK 라인은 카드 안에서 제목선과 여백으로 구획을 나누는 전문가형 테마로 정리한다.
- 구획 배경은 과하게 나누지 않고 흰색 기반 + 얇은 선 + 파란 포인트 라인으로 정리한다.
- PK 라인은 별도 박스 느낌을 줄이고, 여러 줄일 때만 점선으로 자연스럽게 구분한다.

## testV16 반영
- 탭 이름을 열선m계산으로 변경한다.
- PK 라인 제목선은 글자 아래로 내려 제목과 겹치지 않게 한다.
- 구역 설정/배관 정보의 +/− 버튼은 각 영역 제목과 같은 줄 오른쪽에 배치한다.
- 최초 기본 구역/배관에는 +만 보이고, 추가로 생성된 구역/배관에는 +/−가 모두 보이게 한다.
- PK 라인 삭제 버튼은 구역/배관의 +/− 버튼과 같은 크기 톤으로 맞춘다.
